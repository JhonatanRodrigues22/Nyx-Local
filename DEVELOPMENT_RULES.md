# Development Rules

## Architecture

- Never modify architecture without Tech Leader approval.
- The `.ai` directory is official project documentation.

## Code Style

## Testing

## Dependencies

- All development must use the `.venv` virtual environment.

## Documentation

- Every Sprint must start by reading the `.ai` directory.
- Every Sprint must finish by reviewing the `.ai` directory.

## Git Workflow

- No Sprint must be pushed before Tech Leader review.
- The `scripts/prepare_pr.py` script must be used to prepare Sprint branches.
- Automatic push must not happen without explicit JJ confirmation.
- The Pull Request must contain a clear Sprint summary.
- The final commit must follow the title approved by the Tech Leader.

## Delivery

- Generate the clean project package before delivery.
